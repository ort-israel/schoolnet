Moodle question tags - Howto and Best Practices

  • What tags are
  • How tags can be used with quiz questions
  • Tagging best practices
  • Alternatives to tags - Categories and Outcomes (for structured standards)

What tags are

Within the Moodle LMS, various types of objects can be marked with tags. Tags are short keyword labels which tell us something about the object. You may have seen them used on web sites like Flickr or del.icio.us, or you may be familiar with the similar concept of #hashtags from Twitter, Facebook or Instagram. On Flickr, a popular tag is California, indicating a photo of something related to the state of California. Within Moodle, tags can be applied to quiz questions, users, courses, and other items in the LMS. This document discusses using tags with quiz questions.

How tags can be used with quiz questions

Tags can be useful to help label and find questions in a large question bank. For example, if you had a question bank with questions about the the Texas A&M System, some of those questions may specifically be about TEEX. Some questions may be about the history of the system, other questions might be about TEEX history. You could tag those questions TEEX and History to make them easier to find later. Later, when building a quiz based on that question bank, you can easily find all of the questions about TAMUS history by searching the History tag. Tagging best pratices

It is best practice to choose tags that are a) general and b) about the question itself. Keep in mind that in the future someone may want to use your tags to build a quiz for an entirely different course — and that person may be you. For this reason, Medications is a better tag than Chapter 15 - Medications. Two years from now, you may be building a different course and have need for questions about medications, completely unrelated to whatever book "chapter 15" refers to. The key here is that the question itself is about Medications, without reference to any particular course or anything else. The tag on the question tells us something about the question. Of course, this is only a general guideline. You may have a specific need to use tags in a different way. However, when the tag is not about the question itself, consider using Categories or Outcomes instead. Categories and Outcomes are discussed below.

It is recommended that tags be short, most often a single word. Actors rather than Comedic actors of the 1980s. This helps when using the tags later for searching. When looking for actors, you are more likely to just enter the word "actors" rather than come up with the longer phrase. Just as important, it's actually more flexible to use two or three tags, Actors, 1980s & Comedy. In this way, if you later need to find questions about actors, any actors, you can click the Actors tag. If you want only 1980s actors, you can click both Actors and 1980s. Two short tags give you more flexibility than one long tag.

Note in the above examples that tags are always plural, if the plural makes sense. We use Actors, not Actor. This avoids having some items marked in the singular and some in the plural, creating unnecessary extra tags. Use the singular only when plural doesn't make any sense. Plural Barak Obamas doesn't make sense, there is only one Barak Obama, so we'd use the tag Barak Obama.

Alternatives to tags - Question Categories and Outcomes

Categories are sometimes a better tool to use than tags. Categories are used to organize questions into useful groups. Categories can have sub-categories. Categories are best when there is a natural hierarchy, meaning each question is in only one group. For example, categories could be used in a biology question bank to organize questions about Animals -> Mammals -> Primates versus Plants -> Seed Plants -> Conifers. The important thing to note here is that no single item can be both a primate AND a conifer. Categories are good for organizing into exclusive categories of this nature. Categories may be most useful when you want to organize based on something that is not an inherent property of the question itself, but rather represent groupings of questions. As discussed, tags are used to mark properties of a question.

Outcomes are designed to track student competencies in a required curriculum or set of standards. As an example, outcomes can be a better choice than tags when a curriculum standard specifies that students must demonstrate competency in each of a list of skills. An elementary math curriculum might specify that the quiz must include questions covering addition, substraction, multiplication, and division. Outcomes serve this need well. For example, using outcomes you can see the student's average grade for addition questions, average grade for subtraction, etc. Like categories, outcomes are hierarchical, so multiplication and division can be grouped together under "math". Unlike categories, outcomes are not exclusive, so one question could be marked for both the understands regulations outcome and the understands safety rules outcome.

Codified standards are another area where outcomes work well. If, for example, you are developing a course on fire codes, outcomes are an ideal way to track that the course teaches each section of the fire code, and test that students have learned each section of the code. Outcomes should be considered when the student needs to learn each element of a structured, well-defined standard.

Outcomes can also be associated with learning activities within the course. TEEX support for the full outcomes system is planned for the second half of 2014. As of early 2014, a more limited version of outcomes is available. For detailed information about the upcoming outcomes system, see the logical overview and specification documents.
